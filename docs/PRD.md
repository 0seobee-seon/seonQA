## 1. Problem Statement (문제 정의)

선엔지니어링 총무팀으로 직원들의 반복 문의가 집중되어 핵심 업무가 지체되고 있다.  
대부분의 문의는 사내 규정집·양식 문서로 답변 가능하지만, 직원들이 문서를 직접 찾지 않고  
담당자에게 질문하는 패턴이 반복된다.

> **핵심 흐름 1개:** 직원이 질문을 입력 → 사내 문서에서 관련 내용 검색 → 답변 출력

---

## 2. Target User (목표 사용자)

**첫 사용자:** 선엔지니어링 전 직원 (현장 포함)

**실제로 반복되는 업무 문의 유형:**

| 분류 | 대표 문의 |
|------|---------|
| 인사/복무 | 연차 신청, 반차, 출장, 복무규정 |
| IT | 그룹웨어 비밀번호 재설정, 전산기기 신청 |
| 총무 | 사내 전화번호, 문서번호 발급, 렌트차량, 사무가구 |
| 규정 | 급여규정, 인사관리규정, 취업규칙 |
| 양식 | 사직원, 차용증, 연봉변경계약서 |

---

## 3. User Flow (사용자 흐름)

```
1. 직원이 http://[사내주소] 접속
        ↓
2. 질문 입력 또는 상단 빠른 버튼 클릭
   예: "연차 신청 어떻게 해요?" / "홍진욱 내선번호"
        ↓
3-A. 전화/내선/연락처 키워드 포함 → phone_directory.json 직접 조회
3-B. 그 외 → database/ 폴더의 사내 문서(PDF·hwpx)에서 점수 기반 매칭 검색
        ↓
4-A. 관련 내용 있음 → 질문 키워드 주변 발췌 답변 + 출처 문서명 표시
4-B. 관련 내용 없음 → "총무팀 문의" 안내 (이메일·내선 표시)
        ↓
5. 직원이 답변 확인 후 총무팀 연락 없이 스스로 처리
```

## 4. Core Features (핵심 기능)

### F-1. 질문 입력 및 답변 출력 ✅
- 텍스트 입력창 + 전송 버튼 (엔터 가능)
- 빠른 질문 버튼 5개: 연차/휴가 신청, 그룹웨어 비밀번호 재설정, 사내 전화번호 확인, 복무규정 확인, 취업규칙 확인
- 사용자 말풍선(우측) / 봇 말풍선(좌측) 구분
- 로딩 애니메이션(점 3개)

### F-2. 사내 문서 기반 답변 ✅
- `database/` 폴더의 실제 선엔지니어링 문서에서 내용 추출
- 지원 형식: PDF (`pdf-parse@1.1.1`), hwpx (`adm-zip` + XML `<hp:t>` 태그 파싱)
- **점수 기반 문서 매칭:** 파일명 키워드 일치 +10점, 본문 포함 +1점 → 최고점 문서 1개만 반환
- **스마트 발췌:** 질문 키워드가 등장하는 주변 텍스트(±8자, 좁은 창)만 반환 — 문서 전체 반환 방지
  - 문서 파일명에서 뽑은 키워드(문서 선택용)는 발췌 검색에서 제외
  - 겹치는 스니펫(50자 이상 overlap) 제거, 최대 2개까지만 반환
  - 예: "홍진욱 전화번호" 질문 시 "전화번호"는 파일명 키워드라 제외, "홍진욱" 주변만 발췌
- 답변 하단에 출처 문서명 표시
- API 키 없을 때: 문서 발췌 직접 출력 (목업 모드)
- API 키 있을 때: Claude API (`claude-sonnet-4-6`)로 자연어 요약 답변

### F-3. Fallback 안내 ✅
- 문서에서 내용을 찾지 못한 경우 자동으로 총무팀 연락처 안내
- 📧 kimm1027@seon.co.kr / 📞 043-220-8505
- "사람 확인 필요" 판단은 자동 결론으로 표시하지 않음

### F-4. 사내전화 정확도 개선 + Excel 관리 워크플로우 ✅ (신규)
- 전화/번호/내선/연락처/사내전화 키워드가 질문에 포함되면 hwpx 파싱을 거치지 않고
  `database/phone_directory.json`을 직접 조회 (`searchPhone()`)
- 이름·부서·내선번호로 매칭 → 실패 시에만 기존 문서 검색(F-2)으로 폴백
- **총무팀 관리 워크플로우:** `make-phone-excel.py`(openpyxl)로 총무팀이 채울 Excel 양식
  (`사내전화번호_양식.xlsx`) 생성 → 총무팀이 이름/직함/내선/부서/근무지 입력 →
  `node excel-to-json.js`로 `phone_directory.json` 재생성 → 서버 재시작 시 반영
- 담당자가 hwpx 원본을 직접 편집하지 않고 Excel로 전화번호부를 유지보수할 수 있음

| 서버 | Node.js 내장 http | Express 없이 경량 구현 |
